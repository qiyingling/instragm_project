# Instagram_用户行为分析项目

## 项目简介

本项目模拟了 Instagram 社交平台的数据环境，通过 Python、MySQL 和 Tableau 完成从数据生成、数据清洗、数据建模到可视化分析的完整流程。
项目重点关注用户增长、点赞行为、关注关系、内容热度以及用户分层等问题，希望通过构建一个较为完整的数据分析项目，熟悉数据分析岗位常见的工作流程和工具链。

---

## 项目流程

### 1. 数据生成

使用 Python 模拟生成平台数据，包括：
* 用户信息（users）
* 图片信息（photos）
* 点赞记录（likes）
* 关注记录（follows）
* 评论记录（comments）
共生成约 1 万名用户及相关行为数据。

---

### 2. 数据清洗

使用 Pandas 对原始数据进行处理，主要包括：
* 检查缺失值
* 检查重复记录
* 处理异常数据
* 统一字段格式
清洗完成后导出为标准数据文件。

---

### 3. 数据库建模

将清洗后的数据导入 MySQL，建立用户、图片、点赞、关注和评论等基础表。
在此基础上构建分析所需的汇总表，包括：
* 用户增长表
* 点赞活跃度表
* 关注活跃度表
* 用户行为关系表
* 热门内容表
* 用户分层表

---

### 4. 可视化分析

使用 Tableau 构建仪表盘，主要包含以下内容：

#### 用户增长趋势

观察平台用户规模变化情况。

#### 用户点赞活跃度分布

分析用户活跃程度是否存在明显差异。

#### 用户关注行为分布

分析用户社交行为特征。

#### 点赞与关注关系分析

观察用户社交行为与互动行为之间是否存在关联。

#### 热门内容分析

识别获得较高互动量的内容。

#### 用户分层分析

根据关注数量对用户进行简单分层。

## 项目不足

由于项目数据为模拟生成，因此与真实社交平台仍存在一定差距。

### 1. 缺少地区信息

目前用户数据中仅包含基础注册信息，没有包含：
* 国家
* 省份
* 城市
因此无法开展地域分布、区域活跃度等分析。

### 2. 用户关系过于均匀

当前关注关系采用随机生成方式。
而现实中的社交平台通常呈现明显的长尾特征：
* 大部分用户关注数量较少
* 少数用户拥有大量粉丝和互动
当前数据未能充分体现这一特点。

### 3. 缺少用户画像信息

目前没有模拟：
* 性别
* 年龄
* 职业
* 兴趣标签
因此无法进行更细致的用户群体分析。

### 4. 缺少内容分类

图片数据未设置内容类别。
例如：
* 美食
* 旅游
* 体育
* 科技
* 娱乐
无法分析不同内容类型之间的互动差异。

### 5. 行为模式较为简单

当前点赞、关注和评论行为主要依靠随机规则生成。
真实平台中用户行为往往会受到：
* 内容质量
* 社交关系
* 热门事件
* 平台推荐机制
等因素影响。

* Cohort 分析

### 与实际业务场景结合
当前项目主要围绕用户注册、点赞、关注、评论等行为数据展开分析，能够反映平台用户活跃情况和社交关系特征。
但在实际企业环境中，数据分析通常不仅关注用户行为，还需要结合业务目标进行分析，例如增长、留存、转化和收入等指标
